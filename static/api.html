Create Manager Account

Request example:

{
    "password": "mypassword",
    "name" : "Mr Patrick",
    "email" : "some@mymail.com",
    "telephone": "12345678",
}

Response if the username exists in the system:

  • Status code: 403 Forbidden

Response example if successful:

{
    "token": "9944b09199c62bcf9418ad846dd0e4bbdfc6ee4b"
}

and a message like:

Read Notification

This api is for all users to retrieve notifications

Format of data retrieved

[
    {
        "code": 1, // type of notification
        "message": "Scheduling is done",
        "detail": {
            // format varies according to notification type
        } 
    }, 
    ....
]

The meaning of code is as follows:

Code Meaning Targe user
0 Roster Result Out manager
1 Roster arrangement changed (mostly due to swap) manager
2 New time table polling opens to you employee
3 New duty is assigned to you employee
4 Someone sends swap-slot request to you employee
5 Your request for swap is approved employee
6 Your request for swap is rejected employee
7 You are requested to join a new team employee
8 A person you invited has joined your team manager

Common url

For the api that follows the common url pattern, you can do retrieve, create, update in the following way: (let’s say you want to manipulate event data, and the base url for that is /event/)

Create

Request url: /event/ Method: POST For the format of the post data, refer to the detail of the particular api you are using.

If the creation is successful, you will get the created object in json format (same as the format of the post data) and http status code will be 201 created. If there’re some invalid fields in your post data, you will receive something like the following:

{
  "name": [
    "This field must be unique."
  ]
}

The http status code will be 400 bad request

Update

Request url: /event/<object_id>/ Method: PUT, PATCH (To fully update an object, use PUT, and you need to post all the data required for constructing the object) (To partially update an object, use PATCH, and you only need to post the attributes you want to update)

For the format of the post data, refer to the detail of the particular api you are using.

If the update is successful, you will get the created object in json format (same as the format of the post data) and http status code will be 200 ok.

If there’re some invalid/missing fields in your post data, you will receive something like the following:

{
  "name": [
    "This field is required."
  ]
}

The http status code will be 400 bad request

Retrieve

Request url: /event/ Method: GET

By default, it will return a list of all event objects, for the format of the json list, refer to the detail of the particular api you are using.

Http status code will be 200 ok.

To do filtering, you can append query string like the following: /event/?category=Sports&fee=0

Refer to the detail of the particular api to see what filtering fields can be used.
